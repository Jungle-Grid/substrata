# Classification pipeline implementation plan

This plan is ordered to eliminate circular reasoning before adding more product coverage.

1. Add canonical evidence, contradiction, profile-decision, candidate-decision, regulatory-evidence, history-match, provenance-edge, and validated-decision contracts in `workers/classifier/src/evidence_model.py`, `schemas.py`, and `packages/shared/src/schemas.ts`. Preserve legacy fields as a versioned compatibility projection.
2. Add deterministic polarity and configuration-scope normalization. Validate exact source quotations, exclude generated/profile fields from evidence search, make duplicate evidence idempotent, and surface untrusted instruction-like content as warnings.
3. Replace score-only profile routing in `classification_heuristics` with declarative eligibility contracts: required positive source signals, hard exclusions, contradictions, compatible forms, minimum evidence, and explicit `unknown` abstention. Scores rank only eligible profiles.
4. Gate every candidate independently on positive source evidence, product form, blocking contradictions, and regulatory availability. Keep incomplete paths separate from substantiated candidates.
5. Build a canonical `ClassificationDecision` after routing and validate cross-stage invariants before memo generation. Change `memo.py` to consume this object and reject any fact/profile/candidate not in it.
6. Split `history-retrieval.service.ts` into source-fact primary retrieval and labeled hypothesis retrieval. Remove titles, filenames, generated profiles, review prose, questions, and candidate ECCNs from the primary query. Add polarity/configuration-aware reranking and structured `precedent | contrast | irrelevant` explanations.
7. Add a versioned regulatory-source adapter and persisted corpus metadata. Until a synchronized corpus is available, mark evaluation `unavailable` and cap candidate eligibility rather than fabricating current text.
8. Normalize every provider into the same strict extraction contract; log prompt/model/schema hashes and explicit fallback events; run identical deterministic validation after every response. Add prompt-injection separation and provider conformance tests.
9. Add Prisma migration fields for pipeline/rule/prompt/corpus/retrieval/decision versions and first-class JSON evidence/contradiction/decision snapshots. Backfill only version labels for old runs; never reinterpret old decisions.
10. Update API presenters and reviewer UI to distinguish source evidence, derivation, contradiction, internal precedent, regulation, rejected paths/candidates, and abstention. Label history as “Internal precedent only — not regulatory authority.”
11. Add fictional category-diverse fixtures and unit, counterfactual, metamorphic, property, prompt-injection, provider-conformance, integration, and regression tests. Add an evaluation harness for precision, unsupported candidates, abstention, contradictions, provenance, history relevance, citations, and cross-provider consistency.
12. Run worker tests, API unit/integration tests, tenant-isolation regressions, lint, typecheck, build, and desktop/tablet/mobile visual inspection. Record failures and limitations without weakening invariants.
