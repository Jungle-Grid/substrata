# AMD integration

## Summary

AMD-backed execution is a prototype and hackathon-oriented direction for accelerating structured document analysis. It is not a claim of production AMD deployment.

## Why Substrata needs compute

Large technical documents require extraction, normalization, drafting, and batch processing. Compute is useful when teams need faster analysis across many source packages while preserving evidence and review controls.

## AMD-backed execution use cases

The repository includes a Jungle Grid/ROCm-oriented worker-image and runbook path. It is designed to run model-assisted extraction on AMD-capable infrastructure while returning structured output and provenance.

## Document analysis and fact extraction

Accelerated inference can support extraction of device architecture, interfaces, memory, cryptography/security, and performance facts. The deterministic review engine remains responsible for routing and candidate gating.

## Company history batch processing

AMD-backed compute can support future batch ingestion and parsing of private company-history materials. Current retrieval is organization-scoped and lexical; it does not claim a production GPU retrieval service.

## Memo generation

Model-assisted drafting can prepare a review-ready memo draft from structured facts and review paths. Qualified human approval remains mandatory.

## Notebook-driven development path

AMD notebook/manual execution can be enabled as an internal Remote provider for hackathon or demonstration workflows. It is not exposed as a normal end-user choice.

## Future accelerated workloads

Potential work includes resilient remote job submission, provider health checks, artifact import, batch parsing, and measured throughput/cost controls.

## Why this matters

The value is operational: faster source-grounded preparation without reducing reviewer accountability. See the [AMD/Jungle Grid runbook](../hackathon-demo-runbook.md) for prototype preflight details.
